Type Alias

Type aliass are used to declare Java classes to gui4j. For some elements gui4j needs information about the element type (Combobox, e.g.) to support correct typed method calls. In such cases only introduced type aliases using the element ClassAlias are accepted. It can only be declared right after the Include elements.
Note that type aliases might be grouped in a single file xml file (types.xml, e.g.) and included using Include.

<ClassAlias class="java.util.Map$Entry" name="Entry"/>

 The ClassAlias has two attributes:
  • class:
    Use the fully qualified class name. Nested inner classes are written by use of the $ symbol. E.g. java.util.Map$Entry


  • name:
    Choose the name under which you want to use that type in gui4j XML-files.
