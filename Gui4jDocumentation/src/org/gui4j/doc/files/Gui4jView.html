View

View is the root element of each xml file. It supports the two attributes defaultButton and top (see below). Inside this element all relevant information for the graphical user interface is defined.

View Content

Inside the View element the following xml elements might be used several times. Please note that the list below defines the correct ordering of the elements.
  • Param:

    • Define parameters to be subsituted when including this file.

  • Include:

    • Include other xml files to structure large descriptions and to divide descriptions for different controllers.

  • ClassAlias:

    • Introduce a new alias for a Java class. Note that only these aliases can be used for referencing types.

  • Style:

    • Define a new style. A style specifies default values for elements. When using the style these default values are implicitly used for those attributes that are not not defined. These technique allows to build several views with the same look and feel.

  • Component:

    • Defining a gui component.

View Attributes

  • defaultButton

    • The value of this attribute is the name of a xml button definition (guiId) to be used as the default button for the view. The default button is invoked when the user presses Enter. For instance:

    <View defaultButton="closeButton">
      ... some defintions
      <button guiId="closeButton" text="'Close'" actionCommand="closeWindows">
    </View>

  • top

    • The value of this attribute specifies the name of a xml definition which is used as the top level widget (the root) of the window. If the attribute is not defined, then there must be defition with the name (guiId) of TOP.

    Using the top attribute
    <View top="main">
      ... some defintions
      <tableLayout guiId="main" cols="..." rows="...">
        ... some defintions
      </tableLayout>
      ... some defintions
    </View>

     Using the top component
    <View>
      ... some defintions
      <tableLayout guiId="TOP" cols="..." rows="...">
        ... some defintions
      </tableLayout>
      ... some defintions
    </View>
  • title

    • The value of this attribute is a method call for the title of the window. This title is only used if the title parameter provided in the methods to create windows is null

    Using the title attribute
    <View title="'My Title'">
      ... some defintions
    </View>
  • name

    • The value of this attribute is a method call for the component name of the created window

    Using the name attribute
    <View title="getWindowTitle" name="getWindowName">
      ... some defintions
    </View>
